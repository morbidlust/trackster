Trackster

In this project, you will build a music search app using the MusicGraph API. The app has a text input field and a search button. When a user inputs some text and clicks the button, the app will search MusicGraph's database for songs related to that text. When the API request is complete, the app displays each song found in a list. The user can then sort the list in a number of ways and listen to a sample of each song via a play button.

It may feel daunting to imagine all the code to get this to work. When you encounter a complex set of actions, it's a good idea to break down the problem into smaller, simpler parts. Imagine how you might do each piece. Then imagine fitting them together. Try your ideas out. Don't be afraid to fail. The steps in this project are one example of breaking the problem down.

On the other hand, if you feel comfortable attacking the problem by yourself, do not let the steps in this project hold you back. The steps provided are just one way to succeed at this project.

You should complete this project locally on your own computer. Be sure to consult the design spec for information about fonts, colors, and dimensions. You can preview a working version of the website here. This project will rely on material you've previously learned and material you will have to look through documentation to learn.

You will build the app one step at a time. For each step, you will build a section of HTML and CSS using Bootstrap to efficiently structure the layout. Then, you will use JavaScript and jQuery to add functionality to that section. It may not be clear how each piece of the puzzle fits together until the end, but when you are finished you will be proud of your fully functional app!

Build the App Header
3.

Write the HTML and CSS to construct the header of the page. Refer to the design spec as you build everything on the deep purple background, rgb(35, 2, 63).

The app skeleton already has a link to the Bootstrap CDN. Use one Bootstrap container and one row for the entire header.

4.

Build and style the three elements in this section: the title text, search input box, and search button. The header should span seven of the twelve columns with a single column offset, the input field should span two columns, and the search button should span one column. There is one unused column remaining.

The icon for the search button is in the Font Awesome library and is called "search." Information on using Font Awesome icons can be found here.

Add Input Functionality
5.

In trackster.js, use jQuery's .click() method to handle the user clicking on the search button. Add an ID to the button to simplify selecting it with jQuery. Remember, add event handlers when the document is ready. Add this ready handler at the top of the file, outside the previously defined functions.

To check that the event handler works before continuing, use console.log() with a simple string in the click handler function. Open Chrome DevTools and click your button to watch it work.

6.

Now you will replace your log message with the value in the text box. Remove the string you passed into console.log() in the last step. In its place, use jQuery to select the input box and use the .val() method to access the input text.

Refresh the browser, type in some text, click the button, and you should see the written text in the log. Now change the text and try again. The new log should reflect the change.

7.

Now that you have confirmed for yourself that these click handlers work, replace the call to console.log() with a call to Trackster.searchTracksByTitle(). Don't change the argument passed in, it should still be the value of the input box. In the next few steps, you will implement this currently empty function.

Obtain an API Key from Last.fm
8.

To use the Last.fm API, you'll first need to create an account.

After signing up, you'll receive an e-mail from Last.fm asking you to confirm your e-mail address. Click the link in the e-mail to activate your account. Next, login to Last.fm using your credentials .

9.

To use the Last.fm API, you'll need to locate your personal API key. First, navigate to the Last.fm API documentation.

Click on the link that reads "Get an API Account". Enter an "Application name", leave the rest of the fields blank, and click "Submit".

You'll be redirected to a page that provides you with an "API key". This is your personal API key. Take note of your API key either by writing it down or saving it somewhere safe.

Last.fm does not allow you to look up application accounts, so if you lose your API key and need another one, you'll need to complete the instructions listed in this step all over again.

Stuck? Get a hint

Your application key should resemble the following:

z3303p90962e8a5ebx5a2f990a69p448

Note, the API key above is an example and is not valid.

Query the Last.fm API
10.

To get a feel for the Last.fm API, make a request using the example URLs in the documentation.

Remember, in this project you'll query the API to search relevant tracks, so navigate to the API page on how to search for tracks.

Make a request by clicking here. You should see an error stating that you must obtain a valid API key.

Replace the REPLACE_THIS_TEXT_WITH_YOUR_API_KEY in the URL with your valid API key and press "Enter" (or "Return") on your keyboard.

The request should have been successful! Your browser will display to you the body of data (the response) you'll use to access track data from the API and display in the project. The response may look messy, because it's not HTML or CSS. Instead, the response is a JavaScript object with data about songs relating to the search, "tiny" (or whatever title you inputted).

Stuck? Get a hint

The correct documentation page for track searching is here.

11.

In trackster.js, create a constant called API_KEY and set it equal to your Last.fm API key. You'll concatenate this API key constant when you make requests later on.

12.

When your document is ready, use the $.ajax() method to send a request. Use the URL from the example request in step 11. When the request is successful, log the response to the console.

Stuck? Get a hint

Documentation on jQuery's $.ajax() method is located here.

13.

Now it's time to make a dynamic request based on the user's text input. Move your $.ajax() call into the body of the function Trackster.searchTracksByTitle(). Our next step will be to use this function's one parameter, title, as the input to the API request.

14.

Take a look at the following example API request and note the part that says api_key=z3303p90962e8a5ebx5a2f990a69p448:

http://ws.audioscrobbler.com/2.0/?method=track.search&track=tiny&api_key=z3303p90962e8a5ebx5a2f990a69p448&format=json

To translate this to English, it says, "the API key is 'z3303p90962e8a5ebx5a2f990a69p448'". Your goal is to programmatically change this to say "the API key is the constant API key defined in trackster.js".

Remove the API key after the equals sign (=) and use string concatenation (+) to append the value of API_KEY.

15.

Notice the part of the sample API request URL that says track=tiny:

http://ws.audioscrobbler.com/2.0/?method=track.search&track=tiny&api_key=z3303p90962e8a5ebx5a2f990a69p448&format=json

To translate this to English, it says, "the track title query is 'tiny'". Your goal is to change this to say "the query is the track title the user wants to search".

Remove everything after the last equals sign (=) and use string concatenation (+) to append the value of title. Voila!

Check your progress. Save the file in your text editor, refresh your browser, type in some input, click the button, and view the log. Does the information reflect your new term?

Display the Song List
16.

Now that your request returns a dynamic list of songs, it's time to present that list to the user. Begin with the column headers by adding a new Bootstrap row and styling it the bright purple color in the design spec. Fill in this row with the column titles: "Song", "Artist", "Album Art" and "Listeners".

17.

Make another row below the one you created in the last step. Make a Bootstrap container within it to house your list of songs. You will select this container with jQuery later, so give it an ID now to make that simple.

18.

Within this track list container, create an example row of fictional track data. You will use this fictional data like a blueprint to guide construction of the HTML, CSS, and, later, JavaScript. Look in the design spec for example data.

Making fictional data for a dynamic feature like this is called mocking. Before you move on, use the mock to guide you as you write the HTML and CSS for this section. Write the code realistically as if this were going to be a static page containing this one track row.

Soon you will use this mock blueprint in JavaScript to automatically build many identical rows.

19.

To make the "play" button, use the icon named "play-circle-o" from Font Awesome. Make the play button a link to this example URL.

20.

Now you will begin to fill out the JavaScript method Trackster.renderTracks(). This method takes a single parameter, an array of track data (the array that is currently being logged in Trackster.searchTracksByTitle()). Write a loop that iterates through each item of this list. You will fill in this loop next.

21.

Copy the row of HTML for the single, mock song you created and paste it into the body of the loop you wrote in the last step. Wrap this HTML in quotes to make it a valid JavaScript string. Assign this string to a new variable.

22.

The string you made in the last step is a very long line of code. To make it easier to read, break this line up over multiple lines. You can use string concatenation (+) to break up the string and to structure it the way it should appear in index.html. You can add spaces within the string to properly indent the nested HTML.

Stuck? Get a hint

The body of the loop should contain a line that looks something like:

var htmlTrackRow =
  '<div class="row track">' +
  // ...
  '</div>';
23.

Before the loop closes, select the track list container and use jQuery's .append() to add this new track HTML to the list of tracks.

24.

The HTML you're building with jQuery has all the same data. If you tried to type into the search box and search the API right now, the track list would fill in for each piece of returned data! Unfortunately, each track would be the exact same as the mock.

Use string concatenation (+) to replace your mock data in this string with the track data. For example, if your loop's iterator is i, the track's title may be accessed with tracks[i].title. Use console.log(tracks[i]) to discover the other value names.

25.

The key that contains the album art value (a URL to an image) is nested deep in the response body. It's recommended that you access this URL outside of your htmlTrackRow variable (or whatever you named it in Step 23) to keep your code easy to read.

The Last.fm API provides four sizes for album art images: small, medium, large, and extra large. Use the medium size album art.

Create a variable called mediumAlbumArt and set it equal to track.image[1]["#text"].

image represents the array of album art images. [1] accesses the medium size album art image. Finally, ["#text"] is the key that stores the URL value to the album art image.

Make sure to update your htmlTrackRow variable (or whatever you named it) with mediumAlbumArt.

26.

You now have a function to display the track list, but you will not see it work until the function is called from somewhere.

Change the success function of your API request to call Trackster.renderTracks() with the list of tracks as an argument.

Stuck? Get a hint

The list of tracks is an array in the response value. It can be accessed with response.results.trackmatches.track.

27.

Try searching one term, and then another. If you've followed these directions closely, you'll notice something undesirable. The results from the second search are added to the end of the first list.

Use jQuery's .empty() method to clear out the track list before new tracks are added.

28.

Rerun the test suite with TracksterTests.runAllTracksterTests() in Chrome DevTools. They should all pass now.

Look at your fantastic achievement! You have a functional web page that searches an API based on input and displays the results nicely. You are a rock star!

Next Steps
29.

This next set of steps are loose descriptions of ideas to take your music search app even further. You've already accomplished a lot, so be proud. We didn't supply tests for these because how you proceed is your choice.

Before you read the following suggestions, take a moment to think about what you might do next to improve your app.

30.

The design spec only uses a small portion of the data for each track which the API supplies. Add columns for more information. The list might get wide enough to scroll horizontally.

31.

The numbers in the "Listeners" column can be hard to read. For example, numbers like 50421 and 1029333 could benefit from an improvement in their numerical formatting.

Use a JavaScript library to improve the numerical formatting of this column.

32.

On many apps, when a user finishes typing in a search box they can use the "Return" key, a.k.a. "Enter", to submit their search. Implement this in your app.

Stuck? Get a hint

Use the .keydown() event handler. Then you will have to figure out which key the user is pressing.

33.

Animate the title text, "trackster", between the time the user clicks the search button and the time that the song list is displayed.

Stuck? Get a hint

Add a class to the title text. Declare a fun CSS animation for that class. Remove the class from the title text. When you make the API request in JavaScript, add the class to the title element. When the response is received, remove the class.

34.

Allow the user to sort the columns by any attribute of the track information by clicking on the corresponding column.

Stuck? Get a hint

Use data attributes and jQuery's .data() method to retain information after you've used it to build the track list. Then use the JavaScript array .sort() method. You'll have to compare each element to each other by the attribute. Use the JavaScript string .localeCompare() method to compare strings.
